# 大纲规划（outline-planning）

当用户要"列大纲 / 规划剧情 / 设计章节结构"时使用本技能。

## 工作流程（先读后写，渐进式）

1. **先读已有设定，不要凭空臆造。** 依次调用以下工具，把当前项目已沉淀的设定读进上下文：
   - 角色：用角色相关工具列出 / 读取 characters（人物动机、关系、当前处境）。
   - 世界观：读取 worldview（地理、势力、规则、禁忌）。
   - 时间线：读取 timeline（已发生的关键事件、时间锚点）。
   若上述任一类设定缺失或过于单薄，先与用户确认或建议补全，再继续。

2. **明确本次大纲的范围与目标。** 跟用户对齐：是整部书的总纲，还是某一卷 / 某几章？核心冲突、主题、目标字数与节奏是什么？

3. **产出结构化大纲。** 按"卷 → 章 → 关键场景"分层罗列，每一章至少写清：
   - 本章目标（推进哪条主线 / 支线）
   - 出场人物与其当前诉求
   - 核心冲突与转折点（钩子）
   - 与已有时间线 / 世界观的一致性校验

4. **保持与设定一致。** 任何与已读取设定冲突的情节，要么修正情节，要么显式提示用户需要更新设定，**不要静默制造矛盾**。

## 输出要求

- 用中文、条目化呈现，层级清晰（卷/章/场景）。
- 每章给一句话"看点"，便于后续 chapter-writing 技能据此成文。
- 大纲完成后，主动询问用户是否要据此进入某一章的正文写作。
