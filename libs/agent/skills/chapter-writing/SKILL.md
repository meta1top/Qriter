# 章节写作（chapter-writing）

当用户要"写这一章 / 续写正文 / 把大纲变成成稿"时使用本技能。

## 工作流程（先读后写，渐进式）

1. **先读设定与大纲，对齐上下文。** 动笔前依次读取：
   - 本章对应的大纲条目（目标、出场人物、核心冲突、看点）。
   - 出场角色档案：用角色相关工具读取 characters，确认每个人物的性格、说话口吻、当前关系与诉求。
   - 世界观 worldview：本章涉及的地点、规则、势力背景。
   - 时间线 timeline：本章在整体时间线中的位置，避免与已发生事件矛盾。

2. **拆场景再成文。** 把本章拆成若干场景（场景 = 一个连续的时空单元）。逐场景写：
   - 场景目标与情绪基调。
   - 以人物动机驱动对话与行动，**保持各角色既定口吻**。
   - 适度的环境 / 感官描写，服务于氛围，不堆砌。

3. **一致性自检。** 成文后回看：人物行为是否符合其性格？是否引入了与世界观 / 时间线冲突的细节？若发现新设定（新人物、新地点、新事件），提示用户是否需要回写到对应设定中。

## 输出要求

- 用中文写作，正文连贯成段，避免大纲式罗列。
- 控制单章篇幅与节奏，结尾留钩子衔接下一章。
- 写完后简要说明：本章推进了哪条线、埋了哪些伏笔，便于用户决策下一步。
